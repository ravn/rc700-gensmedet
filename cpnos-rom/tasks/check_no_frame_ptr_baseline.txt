# Known SDCC IX+/IY+ frame-pointer violators -- baseline for the
# check_no_frame_ptr.py audit gate.
#
# Each line: <file>.s:<function>:<count>
#
# These are accepted as technical debt because eliminating them
# requires either splitting the function into smaller pieces or
# rewriting hot paths in inline asm.  Both options carry their own
# risk.  Track via a task; do not silently grow this list.
#
# When a function on this list is refactored to fit in registers,
# remove or update its line.  The gate will WARN if the actual
# count is below the baseline (so a successful fix is visible).

# relocator.c::relocate -- header magic check + BSS-pair iteration
# + word-additive checksum loop + dual-header memcmp, all with
# simultaneously-live locals SDCC spills to IX-relative stack.
# Rewriting in inline asm defeats the point of a shared C source.
# See plan #19 + dual-header-plan-2026-05-08.md.  Tracked as task #27.
relocator.s:relocate:10

# resident.c::delete_line -- two ix-relative spills.  Likely fixable
# by simplifying the per-row scroll/clear path; revisit when touching
# the resident display code.
resident.s:delete_line:2

# resident.c::insert_line -- two ix-relative spills.  The row pointer
# (CELL(0, cury)) must survive across the __builtin_memmove call so
# the trailing __builtin_memset can reuse it; SDCC spills it to the
# IX frame.  Recomputing row after the call avoids the spill but
# duplicates the cury*SCRN_COLS shift-add chain (~30 B regression).
resident.s:insert_line:2

# snios_c.c::try_send_frame, try_recv_frame -- CP/NET 1.2 wire-protocol
# state machines (#75 Phase 5+6, plain-C rewrite against
# CPNET_WIRE_PROTOCOL.md).  Multiple simultaneously-live locals
# (running checksum, byte counter, response byte, retry counter) that
# SDCC's local-only iCode allocator can't all keep in registers.
#
# Phase 62 (2026-05-10): try_recv_frame raised from 2 -> 10 after the
# timeout-folding source-level optimization.  Folding the explicit
# `if (r >= 0x100) return RC_RETRY` early-bail into the (r & 0x7F)
# != X check eliminates one branch per recv site -- clang loved it
# (-44 B), but it also collapsed the per-site live-range boundaries
# that SDCC was using to recycle registers, pushing 8 more spills
# onto the IX-frame.  User-authorized loosening (2026-05-10).
snios_c.s:try_send_frame:2
snios_c.s:try_recv_frame:10
